<div class="container">
    <header>
        <h1>面向面试的完整流程解析</h1>
        <p class="subtitle">整合技术要点与面试高频考点的回答框架，结合 OSI七层模型与核心流程</p>
    </header>

    <div class="card animate-card" style="animation-delay: 0.1s">
        <h2><span class="step-number">1</span>DNS解析与缓存</h2>
        <h3>流程</h3>
        <ol>
            <li><strong>浏览器缓存</strong> → <strong>系统缓存</strong>（hosts文件） → <strong>路由器缓存</strong> → <strong>本地DNS服务器（ISP）</strong> → 若未命中，触发递归查询：<strong>根域名服务器 → 顶级域名服务器（.com） → 权威域名服务器</strong>，最终获取IP地址。</li>
            <li><strong>DNS预解析</strong>：通过 <span class="highlight">&lt;link rel="dns-prefetch"&gt;</span> 提前解析关键域名，加速后续请求。</li>
        </ol>
        <h3>缓存失效</h3>
        <p>TTL（生存时间）到期或手动清除缓存时重新查询。</p>
    </div>

    <div class="card animate-card" style="animation-delay: 0.2s">
        <h2><span class="step-number">2</span>TCP三次握手（传输层）</h2>
        <h3>核心作用</h3>
        <p>确认双方收发能力，防止历史连接干扰（如过期SYN报文）。</p>
        <h3>三次步骤</h3>
        <ol>
            <li>客户端发送 <span class="highlight">SYN=1 + Seq=X</span> → 服务器返回 <span class="highlight">SYN=1 + ACK=X+1 + Seq=Y</span> → 客户端确认 <span class="highlight">ACK=Y+1</span>。</li>
        </ol>
        <h3>比喻</h3>
        <ol>
            <li>客户端："我要连你"（SYN） → 服务器："收到，同意连接"（SYN-ACK） → 客户端："好的，开始传数据"（ACK）。</li>
        </ol>
    </div>

    <div class="card animate-card" style="animation-delay: 0.3s">
        <h2><span class="step-number">3</span>HTTP请求与缓存策略（应用层）</h2>
        <h3>请求阶段</h3>
        <ol>
            <li><strong>强缓存</strong>：<span class="highlight">Cache-Control: max-age=3600</span> 直接使用本地缓存（状态码200 from cache）。</li>
            <li><strong>协商缓存</strong>：<span class="highlight">If-Modified-Since</span>（时间戳）或 <span class="highlight">If-None-Match</span>（ETag哈希）验证资源更新（状态码304）。</li>
        </ol>
        <h3>HTTPS扩展</h3>
        <p>TLS握手建立加密通道（非对称加密交换密钥 → 对称加密传输数据）。</p>
    </div>

    <div class="card animate-card" style="animation-delay: 0.4s">
        <h2><span class="step-number">4</span>浏览器解析与渲染（关键步骤）</h2>
        <ol>
            <li><strong>构建DOM树</strong>：
                <ul>
                    <li>HTML解析器将字符流转换为Token，生成树状结构（自动补全未闭合标签）。</li>
                </ul>
            </li>
            <li><strong>构建CSSOM树</strong>：
                <ul>
                    <li>解析CSS生成样式规则树，<strong>阻塞渲染</strong>直到CSSOM完成。</li>
                </ul>
            </li>
            <li><strong>合成渲染树</strong>：
                <ul>
                    <li>合并DOM与CSSOM，过滤不可见节点（如 <span class="highlight">display: none</span>）。</li>
                </ul>
            </li>
            <li><strong>布局与绘制</strong>：
                <ul>
                    <li><strong>布局（Layout）</strong>：计算节点位置与尺寸（可能触发<strong>回流/重排</strong>）。</li>
                    <li><strong>绘制（Painting）</strong>：分层渲染（GPU加速）→ 合成显示。</li>
                </ul>
            </li>
        </ol>
        <h3>JS阻塞</h3>
        <p>默认阻塞DOM解析，可通过 <span class="highlight">async</span>（异步加载执行）或 <span class="highlight">defer</span>（延迟执行）优化。</p>
    </div>

    <div class="card animate-card" style="animation-delay: 0.5s">
        <h2><span class="step-number">5</span>TCP四次挥手（传输层）</h2>
        <h3>流程</h3>
        <ol>
            <li>客户端发送 <span class="highlight">FIN=1</span> → 服务器返回 <span class="highlight">ACK</span> → 服务器发送 <span class="highlight">FIN=1</span> → 客户端确认 <span class="highlight">ACK</span>。</li>
        </ol>
        <h3>核心逻辑</h3>
        <ol>
            <li><strong>TIME_WAIT状态</strong>：客户端等待 <span class="highlight">2MSL</span>（约60秒），确保服务器收到最终确认，防止残留报文干扰新连接。</li>
        </ol>
    </div>

    <div class="card animate-card" style="animation-delay: 0.6s">
        <h2>OSI七层模型映射</h2>
        <div class="table-container">
            <table>
                <thead>
                    <tr>
                        <th>流程步骤</th>

                    </tr>
                </thead>
                <tbody>
                    <tr>
                        <td>DNS解析</td>
                        <td>应用层</td>
                        <td>DNS（UDP）</td>
                    </tr>
                    <tr>
                        <td>HTTP请求/响应</td>
                        <td>应用层</td>
                        <td>HTTP/HTTPS</td>
                    </tr>
                    <tr>
                        <td>TCP握手/挥手</td>
                        <td>传输层</td>
                        <td>TCP</td>
                    </tr>
                    <tr>
                        <td>IP寻址</td>
                        <td>网络层</td>
                        <td>IP、ICMP、ARP</td>
                    </tr>
                    <tr>
                        <td>数据帧传输</td>
                        <td>数据链路层</td>
                        <td>以太网、Wi-Fi</td>
                    </tr>
                    <tr>
                        <td>物理信号传输</td>
                        <td>物理层</td>
                        <td>光纤、电缆</td>
                    </tr>
                </tbody>
            </table>
        </div>
    </div>

    <div class="card animate-card" style="animation-delay: 0.7s">
        <h2>面试高频优化点</h2>
        <ol>
            <li><strong>关键渲染路径优化</strong>：内联关键CSS、延迟非关键JS、预加载关键资源。</li>
            <li><strong>TCP性能</strong>：复用连接（HTTP/2多路复用）、减少握手次数（Keep-Alive）。</li>
            <li><strong>缓存策略</strong>：合理设置 <span class="highlight">Cache-Control</span> 和 <span class="highlight">ETag</span>，减少重复请求。</li>
        </ol>
    </div>

    <div class="summary animate-card" style="animation-delay: 0.8s">
        <h2>总结回答模板</h2>
        <div class="quote">
            "从输入URL到页面渲染主要经历：<br><br>
            1. <strong>DNS解析</strong>（本地缓存 → 递归查询IP）；<br>
            2. <strong>TCP三次握手</strong>建立可靠连接；<br>
            3. <strong>HTTP请求</strong>（强缓存/协商缓存）与响应；<br>
            4. <strong>浏览器解析</strong>生成DOM/CSSOM树 → 合成渲染树 → 布局绘制；<br>
            5. <strong>TCP四次挥手</strong>关闭连接。<br><br>
            优化重点包括：减少DNS查询、复用TCP连接、缓存策略与关键渲染路径优化。"
        </div>
        <p><strong>关联OSI模型</strong>：可将流程映射到应用层（HTTP/DNS）、传输层（TCP）、网络层（IP）等层级，体现分层设计思想。</p>
    </div>
